# Runtime Context

The runtime context is the shared object created by the kernel.

It is how runtime parts communicate without relying on ambient globals.

## What it contains

The current `RuntimeContext` contains:

- `id`
- `state`
- `adapters`
- `controllers`
- `services`
- `events`
- `scheduler`

```ts
const ctx = kernel.getContext();

const phase = ctx.state.getSnapshot().phase;
const loading = ctx.adapters.get('loading');
```

## Lifetime

The context belongs to one kernel instance. It should not be treated as a process-wide singleton.

Adapters and controllers receive the same context during setup, lifecycle hooks, event handling, and disposal.
