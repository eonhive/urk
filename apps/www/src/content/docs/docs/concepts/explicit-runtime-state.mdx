# Explicit Runtime State

URK runtime state is intentionally small.

It records the runtime phase, an optional reason, and the update time.

```ts
const snapshot = kernel.getState();

console.log(snapshot.phase, snapshot.reason);
```

## Current phases

- `boot`
- `loading`
- `ready`
- `transition`
- `paused`
- `error`

## Runtime state is not product state

Runtime state should explain the live runtime surface. It should not become the product database or the application business state store.

Use it to keep lifecycle and operational transitions visible.

```ts
ctx.state.setPhase('ready', 'loading-flow:complete');
```
