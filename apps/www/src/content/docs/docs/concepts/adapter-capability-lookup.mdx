# Adapter Capability Lookup

Adapters are resolved through the runtime context.

Lookup can be optional or required.

```ts
const optionalLoading = ctx.adapters.get<LoadingAdapterApi>('loading');
const loading = ctx.adapters.require<LoadingAdapterApi>('loading');
```

## Why lookup is explicit

Not every runtime has every capability. Explicit lookup makes missing dependencies visible during development and in examples.

`require()` should be used when a controller cannot run without the capability. `get()` should be used when behavior can degrade safely.

## Missing capabilities

Required lookup throws if the adapter is missing.

That failure is useful. It exposes an incorrect runtime composition instead of letting the browser surface fail silently later.
