# Integration Wrappers

Integration wrappers connect URK to framework lifecycles.

They should not redefine the kernel, adapter contracts, controller lifecycle, runtime state, or event model.

## Current packages

- `@urk/react-urk` is the React integration package.
- `@urk/next-urk` is the Next.js integration package.

Both are publishable experimental wrappers. Treat them as downstream helpers around the standalone runtime.

## Wrapper boundary

A wrapper may own framework-specific mount and cleanup mechanics. It should not own core runtime semantics.

```text
framework component -> host element -> URK kernel -> adapters/controllers
```
