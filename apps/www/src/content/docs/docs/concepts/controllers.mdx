# Controller Lifecycle

Controllers orchestrate runtime behavior.

They sit above adapters and runtime state. They react to events, drive loading transitions, coordinate browser surfaces, and keep feature flows understandable.

## Lifecycle

Controllers can participate in:

- `init`
- `start`
- `update`
- `onStateChange`
- `dispose`

## What controllers should not become

Controllers should not become:

- unstructured service blobs
- hidden product-business-logic containers
- replacements for explicit runtime state

They are orchestration units, not the entire application.
