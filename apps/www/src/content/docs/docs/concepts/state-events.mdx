# State and Events

URK keeps runtime state and events explicit.

This page preserves the original concept route. For the canonical split, read [Explicit runtime state](/docs/concepts/explicit-runtime-state/) and [Event routing](/docs/concepts/event-routing/).

## Runtime state

The runtime state model starts small:

- `boot`
- `loading`
- `ready`
- `transition`
- `paused`
- `error`

## Events

Events make runtime behavior inspectable.

They can describe:

- capability changes
- controller actions
- lifecycle transitions
- browser-surface activity

<CodeBlock
  title="Event routing foundation"
  language="ts"
  code={`ctx.events.emit({
  type: 'runtime:phase-changed',
  source: 'state',
  payload: { next, previous },
  timestamp: Date.now(),
});`}/>

## Why this is useful

Explicit state and events are valuable for:

- debugging
- docs examples
- interactive demos
- platform tooling
- runtime inspection

They are part of why URK is a runtime kernel instead of just another state helper.
