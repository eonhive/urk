# Adapter registration

Status: current.

This example shows how a runtime registers loading and storage capabilities, then resolves them through the adapter registry from one focused controller.

Open the runnable example: [Adapter Registration](/examples/adapter-registration/).

## What it teaches

- register loading and storage adapters
- resolve required capabilities with `ctx.adapters.require()`
- check an optional missing capability with `ctx.adapters.get()`
- emit visible events for lookup success, optional miss, and storage write/read
- keep adapter implementation separate from controller orchestration

## Boundary

The example is consumed by the website from private `@urk/examples` internals. It demonstrates current runtime behavior, but the website example package is still private and unstable.
