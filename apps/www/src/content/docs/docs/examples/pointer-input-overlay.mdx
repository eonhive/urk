# Pointer/input overlay

Status: current.

The [Pointer Input Overlay example](/examples/pointer-input-overlay/) connects browser pointer and keyboard input capabilities to visible overlay feedback.

## What it teaches

- bind pointer targets or pointer surfaces
- handle keyboard input
- emit interaction events
- update a visible overlay without coupling to one renderer

## Boundary

The adapters expose browser interaction. The controller coordinates feedback. The overlay stays an example surface, not a renderer-specific input framework.
