# Runtime state panel

Status: current.

The [Runtime State example](/examples/runtime-state/) shows phase snapshots, transition reasons, and state-change observation inside a small public runtime island.

## What it teaches

- read runtime snapshots
- observe phase changes through controller state hooks
- show transition reasons in the state and preview panels
- keep product state out of runtime state while still exposing useful runtime visibility

## Boundary

Runtime state should explain the live runtime surface. It should not become a product database, application store, or framework state replacement.
