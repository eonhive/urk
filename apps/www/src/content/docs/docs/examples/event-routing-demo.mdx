# Event routing demo

Status: current.

The [Event Routing example](/examples/event-routing/) shows events moving from controller emitters through targeted listeners, guarded debug listeners, and the visible runtime event log.

## What it teaches

- emit typed runtime events
- listen to one event type
- use guarded `onAny()` for debug visibility
- keep the event log bounded and readable
- avoid turning every interaction into an event

## Boundary

Runtime events should explain coordination and diagnostics. Direct local behavior should stay direct instead of being routed through events by default.
