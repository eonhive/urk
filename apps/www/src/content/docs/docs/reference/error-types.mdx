# Error types

URK currently uses standard JavaScript `Error` objects for runtime failures.

There is not yet a stable custom error taxonomy.

## Current failure paths

- registering a duplicate adapter id
- registering a duplicate adapter capability
- missing an adapter dependency
- using `require()` for a missing adapter
- using a disposed kernel
- pausing or resuming before boot
- adapter or controller disposal failure

## Runtime error phase

When update or state-change handling fails, the kernel stops the scheduler, emits `runtime:error`, and moves state to `error`.
