# @urk/next-urk

`@urk/next-urk` is a thin Next App Router client-boundary wrapper built on top of `@urk/react-urk`.

Status: experimental, publishable, downstream wrapper.

Package page: [@urk/next-urk](/packages/next-urk/)

## Install

```bash
pnpm add @urk/core @urk/react-urk @urk/next-urk next react react-dom
```

## Public API

- `KernelFactory`
- `useClientKernel()`
- `UrkNextProvider`
- `UrkNextProviderProps`
- all current `@urk/react-urk` exports re-exported unchanged

## Minimal usage

```tsx
'use client';

import { createKernel } from '@urk/core';
import { createLoadingAdapter } from '@urk/adapters/dom';
import { UrkNextProvider, useRuntimePhase } from '@urk/next-urk';

function createAppKernel() {
  return createKernel({
    adapters: [createLoadingAdapter()],
  });
}

function PhaseView() {
  const phase = useRuntimePhase();
  return <div>{phase}</div>;
}

export function App() {
  return (
    <UrkNextProvider createKernel={createAppKernel}>
      <PhaseView />
    </UrkNextProvider>
  );
}
```

## Boundary

Runtime execution belongs in a client boundary. Next.js does not replace the standalone kernel model and should not own server-side runtime execution.

Use `apps/next-proof` as the repo-only proof that this wrapper works in a real App Router app.
