# Getting Started

Start with one runtime surface.

The goal is not to scaffold a whole product shell. The goal is to prove a browser surface with explicit lifecycle, capability registration, runtime state, and event visibility.

## Path

1. [Install the packages](/docs/getting-started/installation/)
2. [Create your first runtime](/docs/getting-started/create-your-first-runtime/)
3. [Register an adapter](/docs/getting-started/register-an-adapter/)
4. [Register a controller](/docs/getting-started/register-a-controller/)
5. [Mount a runtime surface](/docs/getting-started/mount-a-runtime-surface/)
6. [View runtime events](/docs/getting-started/view-runtime-events/)

## Live runtime proof

This docs page can embed a real URK surface without turning the docs shell into the runtime framework.

The internal example mount contract is private and unstable. Public consumers should start with `@urk/core` and `@urk/adapters`.
