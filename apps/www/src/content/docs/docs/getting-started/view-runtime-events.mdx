# View runtime events

Runtime events make live behavior inspectable.

Every `KernelEvent` has a type, timestamp, source, and optional payload.

```ts
const unsubscribe = kernel.getEventBus().onAny((event) => {
  console.log(event.type, event.source, event.payload);
});
```

Use events for runtime coordination that should be visible across the surface. Do not route every local UI state change through the event bus.

## Useful early events

- `runtime:booted`
- `runtime:phase-changed`
- `adapter:registered`
- `controller:registered`
- adapter-specific events such as `loading:changed`

## Cleanup

Unsubscribe event listeners when the surface is disposed.

```ts
unsubscribe();
await kernel.shutdown('surface:dispose');
```
