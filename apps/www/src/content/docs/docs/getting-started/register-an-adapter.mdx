# Register an adapter

Adapters expose capabilities to the runtime context.

The loading adapter is a small first adapter because it has an obvious state surface and emits visible events.

```ts
import { createKernel } from '@urk/core';
import { createLoadingAdapter } from '@urk/adapters/dom';

const kernel = createKernel({
  adapters: [createLoadingAdapter()],
});

await kernel.boot();
```

## Lookup

Controllers and runtime code resolve capabilities through the adapter registry.

```ts
const ctx = kernel.getContext();
const loading = ctx.adapters.require('loading');
```

The registry is explicit. A runtime should not assume every adapter exists.
