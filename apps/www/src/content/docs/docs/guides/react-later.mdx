# Use URK with React

React integration is an integration layer, not the source of URK architecture.

Use the standalone kernel model first. Then wrap mount and cleanup in React when a React app needs the runtime surface.

```text
React component -> host ref -> URK kernel -> adapters/controllers
```

## Current status

`@urk/react-urk` is a publishable experimental wrapper package.

Do not treat React as the primary runtime model. The wrapper consumes an existing kernel boundary.

For package details, read [@urk/react-urk](/docs/reference/react-urk/).
