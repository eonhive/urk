# Use runtime state safely

URK state describes the runtime, not the product.

Use it for lifecycle phase and operational visibility.

```ts
ctx.state.setPhase('loading', 'assets:begin');
ctx.state.setPhase('ready', 'assets:complete');
```

## Safe practices

- use specific reasons
- subscribe only when the surface needs updates
- unsubscribe during teardown
- keep product records outside the runtime store
- do not encode every UI control value into runtime phase

## Subscribe to state

```ts
const unsubscribe = ctx.state.subscribe((next, previous) => {
  console.log(previous.phase, '->', next.phase);
});
```
