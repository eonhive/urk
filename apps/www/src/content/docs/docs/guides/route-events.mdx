# Route events

Events should make runtime behavior visible.

Use them for lifecycle transitions, adapter capability changes, controller milestones, and debug panels.

```ts
const unsubscribe = ctx.events.on('loading:changed', (event) => {
  console.log(event.payload);
});
```

## Naming

Use names that state the source and action:

- `loading:changed`
- `input:key-down`
- `pointer:select`
- `runtime:phase-changed`

## Avoid event sprawl

Do not route simple local changes through global events. Direct calls are clearer when behavior stays inside one small unit.
