# Create a controller

A controller coordinates runtime behavior.

It can read context, use adapter capabilities, update runtime state, emit events, and clean itself up.

```ts
import type { ControllerRegistration } from '@urk/core';

export const exampleController: ControllerRegistration = {
  id: 'example-controller',
  start(ctx) {
    ctx.state.setPhase('ready', 'example-controller:start');
    ctx.events.emit({
      type: 'example:started',
      source: 'example-controller',
      timestamp: Date.now(),
    });
  },
  dispose(ctx) {
    ctx.events.emit({
      type: 'example:disposed',
      source: 'example-controller',
      timestamp: Date.now(),
    });
  },
};
```

## Keep controllers focused

Controllers should orchestrate. They should not contain adapter implementation details, renderer internals, or the entire application business layer.
