# Create an adapter

An adapter exposes one runtime capability.

Keep the adapter focused. It should know how to set up, expose, and dispose a capability. It should not own product orchestration.

```ts
import type { AdapterRegistration } from '@urk/core';

export function createExampleAdapter(): AdapterRegistration<ExampleApi> {
  return {
    id: 'example-adapter',
    capability: 'example',
    setup(ctx) {
      return {
        publish(message: string) {
          ctx.events.emit({
            type: 'example:message',
            source: 'example-adapter',
            payload: { message },
            timestamp: Date.now(),
          });
        },
      };
    },
  };
}
```

## Adapter checklist

- use a clear capability name
- expose a small typed API
- use `isSupported` when the browser capability is optional
- clean up listeners in `dispose`
- emit events when runtime-visible behavior changes
