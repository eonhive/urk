# Bridge scene and User Interface

Scene and User Interface (UI) coordination is a runtime problem when both surfaces need shared visibility.

URK can coordinate that behavior through adapters, controllers, state, and events.

## Pattern

```text
pointer adapter -> controller -> runtime state/event -> UI overlay
```

The scene renderer owns pixels. The UI owns presentation. URK coordinates runtime behavior between them.

## Current status

This guide describes the pattern used by the current [Scene/UI Bridge example](/examples/scene-ui-bridge/). The example is intentionally small and does not turn URK into a renderer.
