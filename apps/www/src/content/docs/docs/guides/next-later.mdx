# Use URK with Next.js

Next.js integration must respect server and client boundaries.

URK runtime execution belongs on the client because it coordinates browser capabilities.

```text
Next.js route -> client host component -> URK runtime surface
```

## Current status

`@urk/next-urk` is a publishable experimental wrapper package, and `apps/next-proof` is the repo-only proof app.

Do not present server rendering as runtime execution. Use the wrapper inside a client boundary.

For package details, read [@urk/next-urk](/docs/reference/next-urk/).
